11) O que é um root server?


Resposta:


Um servidor raiz é um tipo de servidor de computador que responde a solicitações feitas por um computador cliente, como um navegador da web. É o primeiro ponto de contato entre um cliente e a Internet, e é responsável por fornecer ao cliente informações como o endereço IP do servidor que hospeda os dados solicitados. Os servidores raiz geralmente estão localizados em alguns centros de dados em todo o mundo e são constantemente atualizados com as informações mais recentes sobre os domínios e endereços IP de nível superior da Internet. Isso garante que os clientes sempre recebam as informações mais atualizadas ao solicitar dados da Internet.



12) Quem são os componentes do DNS e qual a função de cada um deles?


Resposta:


O Domain Name System (DNS) é um sistema de nomes de domínios para computadores conectados à Internet ou à uma rede privada. É um sistema hierarquizado e distribuído, responsável por traduzir nomes legíveis da Internet em endereços IP. Ele permite que os usuários se conectem aos sites através de nomes amigáveis, evitando que eles usem endereços IP difíceis de lembrar. O DNS é composto por quatro componentes principais: servidores de nome raiz, servidores de nome de domínio, servidores de nome de host e servidores de endereço de host.

13) Quais os tipos de servidores DNS possíveis?


Resposta:


Existem dois tipos principais de servidores DNS: o servidor primário e o servidor secundário. O servidor primário é responsável por todas as operações de DNS, como configuração, administração e resolução de nomes. O servidor secundário é usado para fins de redundância e segurança, caso o servidor primário falhe. Além disso, há também servidores DNS especializados, como servidores de configuração automática, servidores de pesquisa de caminho e servidores de recursos DNS estáticos.


14) Qual a diferença de uma requisição iterativa para uma recursiva?

Resposta:


Uma requisição iterativa é uma forma de realizar um processo através de iterações, ou seja, passando por uma série de instruções. Seu principal objetivo é repetir um conjunto de instruções até alcançar um determinado resultado. Enquanto isso, a recursão é um processo que se repete, onde uma função chama a si mesma. A recursão visa realizar uma tarefa complexa através de uma série de tarefas mais simples. Ela é útil para realizar tarefas que possuem um padrão ou estrutura repetitiva.

15) Quais as principais características do protocolo FTP?

Resposta:


O Protocolo FTP (File Transfer Protocol) é um protocolo usado para transferir arquivos entre computadores na internet, que pode ser usado via comandos de terminal e programas. O FTP Ativo permite que o cliente estabeleça uma conexão com o servidor, selecione um porto e o servidor defina um porto para o canal de transferência de dados. O FTP Passivo é usado mais comumente, em situações onde o servidor FTP não consegue estabelecer conexão (normalmente causado pelo bloqueio de portas por firewall). Esse modo permite que o cliente estabeleça duas conexões: uma para comandos e outra para transferência de dados. 16) Como o FTP utiliza suas conexões no momento de execução?


Resposta:


O File Transfer Protocol (FTP) é um sistema padronizado que permite que computadores se conectem e troquem informações. Um dos principais usos do FTP é a transferência de arquivos entre computadores, como a transferência de dados para hospedagem em servidor web. Ao utilizar o protocolo FTP, o cliente e o servidor estabelecem uma conexão segura e criptografada, geralmente com o protocolo Secure Sockets Layer (SSL). Essa conexão é então usada para transferir os arquivos entre os dois pontos de conexão.


17) Faça um breve resumo do protocolo HTTP.

Resposta:


O protocolo HTTP (Hypertext Transfer Protocol) é um protocolo de comunicação que permite que um servidor web responda a solicitações de um cliente. Ele define como os dados da Web são formatados, enviados, recebidos e armazenados. O HTTP usa métodos, cabeçalhos, URI (Uniform Resource Identifier) e corpos de solicitação / resposta para controlar a transferência de recursos para clientes solicitantes. Ele foi originalmente desenvolvido para permitir a interação de usuários e recursos da Web. Atualmente, o protocolo HTTP é amplamente usado para transferir dados entre servidores e clientes em toda a web.
